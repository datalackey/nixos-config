This document provides general, intentionally vendor-agnostic descriptions of the roles
I typically assume when working with LLMs:

    - developer (DEV)
    - researcher (SEARCH)
    - product marketer (PM)
    - UI designer (UX)

Each role is denoted via a fixed string token identifier which carries through in links
across other LLM directive documents. Sub-roles inherit all aspects of parent roles
unless explicitly stated otherwise, and sub-role names adhere to this pattern:
<TOP-LEVEL-ROLE>-<SUB-ROLE>. Example: top level DEV, sub-role DEV-JS (for
Javascript/TypeScript/Node projects).
